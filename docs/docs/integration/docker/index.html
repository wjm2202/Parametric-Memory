Docker

Run MMPM in Docker with persistent storage.

Quick start

The server starts at http://localhost:3000.

docker-compose.yml

The provided docker-compose.yml runs the HTTP API server (node dist/server.js) with a named volume for persistent storage.

Key points:

  • The container runs the HTTP API server, not the MCP server
  • Data is persisted in a Docker named volume mapped to ~/.mmpm/data
  • Port 3000 is exposed for API access

Connecting MCP to Docker

The Docker container runs the HTTP API only. To use MCP (for Claude Desktop or VSCode), you still run npm run mcp:serve locally — it bridges stdio to the Docker HTTP endpoint:

Environment variables

Set in .env at the project root:

VariableDescription
MMPM_API_KEYAPI key for authentication
MMPM_PORTServer port (default: 3000)
MMPM_DB_PATHDatabase path (default: ~/.mmpm/data)
MMPM_LOG_LEVELLog level: info, debug, warn, error
MMPM_BLOCK_SECRET_ATOMSSet to 1 to reject atoms containing secrets
