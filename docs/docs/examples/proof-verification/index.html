Proof Verification

How to verify Merkle proofs independently.

Overview

Every time you access an atom, MMPM returns a Merkle proof. You can verify this proof independently — no API key required.

How verification works

  1. Hash the atom to produce the leaf hash
  2. Walk the proof path from leaf to root:
    • At each level, combine the current hash with the sibling hash (from the path)
    • The direction tells you which side the sibling is on
  3. Compare the computed root with the claimed root
  4. If they match, the proof is valid — the atom existed in the tree at that version

Multi-agent trust

In a multi-agent system:

  1. Agent A writes an atom and receives a proof
  2. Agent A passes the atom + proof to Agent B
  3. Agent B verifies the proof against the MMPM server
  4. If verified, Agent B knows the atom is authentic and unmodified

No trust assumptions between agents. Cryptographic proof replaces trust.

Performance

Verification takes 0.032ms p95 — fast enough to verify on every read without impacting latency.
